The technique discussed below utilizes beam elements to represent a preloaded bolt.

In an assembled structure, modeling bolts as solid bodies usually results in computationally large models that can take a long time to solve. 

It can be very efficient to model the bolts as line bodies rather than solid bodies,
especially if tens or hundreds of bolts are present.

---

### Preparing the Geometry

When using Beam Connections, we don’t need the geometries for bolts, washers or nuts. However, before deleting/suppressing these bodies, we should prepare geometry for analysis

Imprint the face of the washer/bolt head/nut on the geometry. Although you could tie the vertex of the beam connection to the edge, it is more accurate to tie it to the area representing the region contacting the bolt head/washer/nut. The preload will be distributed on the finite area, which is
more accurate.

### Beam Connections for Bolts
Using beam connections to represent a bolt is a common practice. This method does not require a solid body geometry.

1. To insert a beam element, right click on “Connections”. Then, hover over Insert and select Beam. It will bring up the properties menu shown on the right.  
2. Start by setting the radius of your bolt.
3. Under the “Reference” section, the scope represents the face that the head of the bolt will contact. It does not need to be the washer face, it can be any surface. 
4. By default, the bolt will be placed at the center of the reference face. To redefine this position, define the “Reference Location” as the top edge of your hole.
5. The “Pinball Region” represents a conceptual "sphere of influence" around contact or joint elements that defines a search area for interaction with other elements. You can define it as the diameter of your bolt head.
6. The “Mobile” section, the scope represents the bottom constraint for the bolt. For a threaded hole, you will select the inner face. For a nut connection with a washer, you can select a predefined washed face or the bottom edge (making sure to then define your pin ball radius).

### Setting Bolt Pretention

1. Right clock on “Static Structural” and insert a Bolt Pretention. 
2. For bolts defined by beams, change the scoping method to beam connection.
3. Next select a bolt, it is recommended to rename the beam elements to be unique.
4. Lastly, define your preload.

### Defining Load Steps

We want to apply our pretention and then apply our loads. If we do not define two separate steps here, ANSYS will apply everything simultaneously.

1. Select Analysis Settings under Static Structural
2. Change the number of steps to two.
3. Set Auto Time Stepping to Off.
4. Make sure Define By is set to Substeps.
5. Enter 1 for the number of sub steps
6. Change the Solver Type to Direct.

Now we want to define our two steps in out bolt pretension feature. We will do this by changing the 
tabular data for each feature, found in the bottom right of the window.

1. In the newly create second step, change the “Define By” to be lock.
2. Now in any applied loads, make sure they are set to zero for step one and only applied for step two.
